Breadcrumbs

Description

Breadcrumbs facilitate navigation within a site by providing a retraceable path of pages the user has visited. They are especially useful if a site has various parent and child pages (i.e. a hierarchical structure, so you may have a page that links to another subpage). For example, you may have breadcrumbs that look like so: “Homepage > Parent page > Child page > Section page > Subsection page”.

Placement

Breadcrumbs generally appear after a page’s header and are arranged horizontally in a single line.

Interactions

All of the pages listed in a breadcrumb trail must link to the actual pages, so when the user clicks on a breadcrumb the site takes them to the corresponding page.

Header

Description

In the same way that a header in a Microsoft Word document appears in all pages, a site’s header always displays in all of its pages.

Content

Use this component to establish your brand by including your logo and site name. You may also use the header to provide features you want users to always have access to including a link to log in, account or profile access, and search capabilities, just to name a few. See the image above to see the different components one may include in a header:

  1. Logo and site name
  2. Keyword search
  3. View bookmarks
  4. Notification functionality
  5. Profile picture
  6. Links to users' information
  7. Date
  8. Add bookmark

Placement

Position your header so that it stretches across the top of all of your pages.

Footer

Description

In the same way that a footer in a Microsoft Word document appears in all pages, a site’s footer always displays in all of its pages.

Content

Use this component to post any copyright information, contact information, affiliations to other organizations, and any links to secondary pages that may not be as important.

Placement

Position your footer so that it stretches across the bottom of all of your pages.

Local Menu

Description

A menu that appears in a related set of pages on your site. Continuing with our example for TDC, consider the CSI API pages, which contain a menu that is exclusive to them. A user will only be able to navigate through the CSI API child pages by using this menu.

Placement

A local menu can appear horizontally across the top of the page in a tabbed layout or vertically as a traditional side menu flush against the left side of the page.

Pagination

Description

This component consists of a set of numbered pages that link to a portion of content. If you have too much content to display in one page and that content consists of a series of items you can count (e.g., search results or rows in a table), you need to break it up into sections that contain a set number of items.

For example, if you have a search functionality and your user executes a search that displays over 200 results then it would be appropriate to split up that content so only 100 results are displayed per page; you would provide links to two pages in your pagination.

If you need pagination, you must implement a functionality that allows users to display 20, 100, or all results in a page.




Placement

This component is used within a data grid and should be positioned in the lower right-hand corner of it.

Links used to display a specific number of results should be positioned on the lower left-hand corner of the data grid.

Progress Display

Description

Implement a progress display if you are leading your user through a process; for example, if you are guiding the user through a workflow or different steps. This component allows users to know exactly what steps they have completed and gives them an idea of how much more they have left to complete.

Placement

Position this component at the top of the container holding the form or section that users must complete.

Tree

Description

This is a hierarchy of items consisting of parent and child elements used to facilitate navigation between different pages or data. Parent groups can be in a collapsed or expanded state. Expanded parent items display all of the child elements while collapsed parents hide all of the child elements.

Placement

Trees are generally positioned to the left of the page like a side menu.
