Accordion

Overview

Accordion menus are used with content pages. This is used to navigate to pages and subpages (when necessary). There are three types of pages varying levels on what the content needs.

Design Breakdown

Placement

These menus are left of the main content.


Single Level Accordion (accordion with 1 level)

This menu is used when there are no subpages with the content. The selected page is set to a lighter blue (#0795DD) the rest of the menu is set to the default AT&T Blue.

Dual Level Accordion (accordion with 2 levels)

This menu is used when there’s one level of subpages. When selected both the first menu level is set to a lighter blue (#0795DD) the rest of the first menu level is set to the default AT&T Blue. The second level is highlighted in AT&T Orange and the rest of the second menu level links are set to AT&T Blue.

Tertiary Level Accordion (accordion with 3 levels)

This menu is used when there’s two levels of subpages. When selected the first menu level is set to a lighter blue (#0795DD) the rest of the first menu level is set to the default AT&T Blue. The second menu level the selected is set to a Gray Highlight the rest of the second menu level is set to a slightly darker gray (#e5e5e5). The third level is highlighted in AT&T Orange and the rest of the second menu level links are set to AT&T Blue.

Tables

Tables are used to display data in a tabular format. Data is arranged in a logical manner from most valuable data in the left column. The striped style alternates shading by row which is ideal for records with many columns for easier legibility.

Table - Column Table

Column Table allows for column selection. When users need to select columns to show or hide. Do not double the attributes into one column or cell.

Initially show the most valuable columns for the end user. Whenever possible shorten attribute information to keep column width to a minimum (i.e. for date show 1/2/15 instead of January 2, 2015) If certain table attributes only apply to certain groups of users or certain circumstances, consider providing the users with alternative table views.

The table view selected can become the default view for future application sessions. If horizontal table scrolling is absolutely necessary, keep the primary record column fixed to the left as user scrolls through other columns. You may wish to title your table to improve data context. If so, we recommend that you include a title within a table header, left aligned.

Table - Data Table

In general tables should be filterable, sortable and include pagination. Allow users to select the number of results to select the number of results be shown per page. When sorting the column header is highlighted indicating that it is the item that is defining the sort order.

If table scrolls vertically within its container, keep the column headers fixed in the scroll so that users can contextualize data.

Group rows into categories when convenient for the user (use accordion module).

Upon mouse hover, highlight the row if the table is in grid-outline style.

Use tooltips upon long hover to show column heading. You may wish to title your table to improve data context. If so, we recommend that you include a title within a table header, left aligned. An example of how to style this table component can be found in the Task-Oriented Table module.

Table - Tree Table

Tree tables are used for displaying tabular data that has a consistent hierarchical pattern. Plus and minus icons are used to expand and collapse the child rows.

Include an expand all button at top where expandable information is available.

Tabs

Tabs allow the user to navigate among alternate views within the same context. The selected tab is highlighted visually. Tabs may appear within any container on a page.

Using more than 3 tabs to divide content on mobile device is not user-friendly.

An accordion navigation structure or drop down menu is preferred to show or conceal content views on a mobile device.
