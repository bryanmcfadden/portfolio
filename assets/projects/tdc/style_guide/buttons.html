Buttons

This type of component is commonly used to submit or save information in forms; to provide a means for the user to access more information through a modal; or to allow the user to navigate to another page.

Description

Button styles refers to the sizing and colors used for buttons. We will also discuss the use of special buttons in this section.

The size for buttons can vary to match the size of elements that are positioned closely to them to promote proper grouping. For example, if you have a keyword search text field with a “Search” button, you would want the height of the two elements to closely match each other. This supports a better user experience as it allows the user to quickly associate the components with each other.

In addition to sizing, different colors affect the way users perceive a button as well as their expectations as to what the button may do. For example, when using a button in a form, the most commonly used button or most important button should be colored AT&T Orange; this color draws attention and makes users immediately focus on the element when it is in a group.

Special Buttons

One last aspect of buttons you should be aware of is the special classes that exist within TDC. We currently have three special buttons:

  • Team Chat: opens a chatroom in Q so the user can communicate with his or her team
  • Collaborate: links to a tSpace page, usually a forum
  • Attach Files: typically used in forms to allow users to upload their documents

Sizing

A button's width and height must be at least 42px.

Regardless of the size, your button must have a padding of 10px all around. We do not have a set of standard button sizes since labels and/or icons can cause a significant variation in the width of the button. Please use your best judgment when it comes to your button's dimensions. If you have any doubts, feel free to contact us.

Refer to the table below to see examples of some buttons with different dimensions. You do not have to use the exact dimensions seen below; these buttons are meant to give you a better idea of what some dimensions look like.

Button Labels and Icons

Button labels should be clear and concise. If you find yourself using a rather long label, please consider revising it. Users should get an idea of what the button does after reading the first two words of the label. We suggest you use action-oriented verbiage (e.g. “Create Project” instead of “New Project”) to help the user determine the button’s functionality.

Follow these standards when you include a label and/or icon in your button:

  • The color of the label and/or icon should be white except if the color of the button is AT&T Gray Highlight, in which case the label and/or icon should be AT&T Gray
  • Position the label and/or icon so the component(s) is/are centered both horizontally and vertically within the button
  • You may include no more than one icon in a button

Colors

AT&T Blue (#067ab5)


Used for all single buttons (i.e., buttons that are not grouped with any other buttons) in TDC. This color can also be used for buttons that do not need emphasis when they are in a group.

AT&T Blue Highlight (#0574ac)


When the user hovers over a button colored AT&T Blue it needs to change to AT&T Blue Highlight.


AT&T Orange (#ff7200)


Use this color to place emphasis on a button within a button group.


AT&T Orange Highlight (#fcb314)


When the user hovers over a button colored AT&T Orange it needs to change to AT&T Orange Highlight.


AT&T Gray Highlight (#f2f2f2)


When a button is disabled use this color.


General Placement

When a button is inside of a portlet or container if the button is not associated with an input field, it should be displayed as block and centered horizontally.

If a button or button group is in a form they must be positioned in the lower right corner of the form.

Finally, if the button is associated with an input field, it must be positioned to the right of that element.

Button Group

In TDC, buttons are generally grouped in twos and threes. The number of buttons within a group depends on the number of options you intend to have. Do note that button groups are usually used in forms.
