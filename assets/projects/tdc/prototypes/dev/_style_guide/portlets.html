Portlets

Portlets are containers used to display content and may be used in any type of layout. If you want to present content in an at-a-glance fashion then portlets are your answer. TDC uses two types of portlets: frameless and framed. Both of these portlets follow JSR 256 specifications.

Layouts

You can combine and include any of the following content types in a portlet:

  • Links
  • Text
  • Images

If you only have a list of links and their names are self-explanatory, you can use the layout seen in Portlet 1. There exists another variation of the layout including links with brief descriptions (see Portlet 2). We recommend you only use this if the names of the links listed aren’t clear; for example, if you are linking to an application with which most people are unfamiliar it would be appropriate to include a description. Otherwise, you should consider revising the name of the link to something that is clear and concise. The last type of portlet can have a mixture of content types as seen in Portlet 3.

In addition to the frameless portlet, you may also use framed portlets to display content. If you are pulling content from a data source for which you want a custom display then you should use this type of portlet. Not only will you be able to customize your displays, but you will be able to easily implement filters and multiple views of the data.

Note that these types of portlets have a header that can have a help and expand/collapse functionality among other things. Scrollbars can also be included inside the portlet when lots of content is included. You can learn more about JSR 256 portlets by looking at the specification by Oracle. See “Interactions” to learn more about how these portlets can be used to manage content.

Interactions

JSR 256 portlets can be dynamic. As mentioned before, portlets can have different functionalities in their header. In the example below, a PRISM portlet, the user can get more information about it from the icon in the header. Finally, the user can also sort information within the table and navigate through the different pages to view more information. If you want to display data and allow users a way to customize the way it’s display, use JSR 256 such as this one.
